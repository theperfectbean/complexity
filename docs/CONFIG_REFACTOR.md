# Config Refactor Status

## Current Implementation (Done)
- Added centralized runtime config at `app/src/lib/config.ts` with defaults matching existing behavior. It reads from `env` and exposes config for models, LLM aliasing, Perplexity, RAG, memory, chat limits, uploads, auth, Redis, documents, and embedder.
- Expanded env schema in `app/src/lib/env.ts` to include new configurable knobs and defaults (timeouts, limits, templates, URLs, etc.). This is the source of truth for env parsing.
- Updated models module `app/src/lib/models.ts` to source `MODELS` and default model from `runtimeConfig` instead of hardcoded constants.
- Wired config into runtime code:
  - `app/src/lib/llm.ts`: use `runtimeConfig.llm` for model aliases, Ollama base URL, local OpenAI key fallback.
  - `app/src/lib/perplexity-agent.ts`: use `runtimeConfig.perplexity` for base URL, tools list, timeout.
  - `app/src/lib/perplexity.ts`: use config for stream message id or keep constant if intentional.
  - `app/src/lib/rag.ts`: use `runtimeConfig.rag` for chunking, timeouts, batch size, concurrency, embed path, similarity default.
  - `app/src/lib/memory.ts`: use `runtimeConfig.memory` for TTL/prefix/max, extraction cadence, model, prompt strings, failure prefix.
  - `app/src/app/api/chat/route.ts`: use `runtimeConfig.chat` for rate limit, cache TTL, attachment limit, fallback text, memory timeout.
  - `app/src/app/api/roles/[roleId]/upload/route.ts`: use `runtimeConfig.uploads`.
  - `app/src/lib/documents.ts`: use `runtimeConfig.documents.allowedExtensions`.
  - `app/src/auth.ts`: use `runtimeConfig.auth` for password policy and bcrypt cost.
  - `app/src/app/api/auth/register/route.ts`: use `runtimeConfig.auth.passwordMinLength` and `bcryptCost`.
  - `app/src/app/api/auth/reset-password/route.ts`: use `runtimeConfig.auth` for password min length, bcrypt cost, reset token TTL.
  - `app/src/app/api/auth/forgot-password/route.ts`: use `runtimeConfig.auth` for token bytes, TTL, email subject/body templates, default from, localhost base URL.
  - `app/src/lib/redis.ts`: use `runtimeConfig.redis.maxRetriesPerRequest`.
  - `embedder/main.py`: read env for model name, app title, and version.
- Aligned settings keys:
  - Fixed mismatch between `app/src/app/api/settings/route.ts` and `app/src/app/settings/admin/page.tsx` (added missing Ollama/Local OpenAI keys).
  - Refactored `app/src/app/api/settings/route.ts` to handle both user `memoryEnabled` (via `PATCH`) and admin API keys (via `GET`/`POST`).
- Updated tests:
  - Fixed `app/src/lib/config.ts` to handle undefined env strings during tests.
  - Updated `app/src/app/api/chat/route.test.ts` to mock `@/lib/settings` and fix text extraction from `response.completed`.
  - Added try-catch to Perplexity block in `app/src/lib/llm.ts` for robustness.
